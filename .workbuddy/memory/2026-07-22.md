## 修复空间发帖/聊天发文件 400 错误（FormData Content-Type bug）
- 现象：空间点击发布后没反应、发布框不关闭；好友看不到帖子；状态也有类似问题
- 诊断过程：
  - 线上后端已有 posts 路由（另一个会话已 railway up 部署新版后端）
  - curl 实测：POST /api/posts 带 FormData + `Content-Type: application/json` → 400 "请求格式错误"
  - Railway 日志：用户 GET /api/posts/feed 200，但**完全没有 POST /api/posts 记录**（400 在 express.json() 层被拦截，logger 在它之后所以看不到）
- **根因**：api.ts request() 无条件设置 `Content-Type: application/json`，`posts.create` 传的 `headers: {}` 是空对象展开，不会覆盖默认 Content-Type → FormData 带错误的 Content-Type → 后端 express.json() 把 multipart body 当 JSON 解析失败 → 400
- **影响范围**：posts.create（空间发帖）、messages.send 带文件（聊天发图片/文件）两处 FormData 请求
- 修复（前端）：
  - api.ts request()：检测 `options?.body instanceof FormData` 时跳过 Content-Type 设置，浏览器自动带 multipart/form-data + boundary
  - Space.tsx + StatusPage.tsx：发布失败增加用户可见错误提示（原来 catch 只 console.error，用户"点了没反应"）；发布中按钮显示"发布中..."
- 部署：prebuilt 部署成功（index-CN9CU2wf.js），线上验证含 instanceof FormData 逻辑
