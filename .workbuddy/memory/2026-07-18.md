# 2026-07-18 工作日志

## 探时 textime.top 线上 API 全挂修复

### 问题
用户访问 textime.top，登录/注册全部报"网络问题"，控制台显示 API 请求失败。

### 诊断
- 直连 api.textime.top/api/health → 200 JSON（正常）
- 经 textime.top/api/health（Vercel rewrite）→ 403，cf-mitigated: challenge，返回 Cloudflare "Just a moment" JS 质询页 HTML
- 根因：api.textime.top 在 Cloudflare 开橙云代理 + Bot 防护(Managed Challenge)。Vercel 服务器端 rewrite 用数据中心出口 IP，被 CF 判定为机器人返回 JS 质询页；服务器端无法执行 JS → 所有 /api 拿到 HTML → 前端 api.ts 报"网络连接失败"

### 修复（用户选择"改代码直连"方案）
改动 3 个文件 + 修复 1 个构建隐患：
1. src/utils/api.ts — API_BASE 生产环境默认直连 https://api.textime.top，并 export
2. src/App.tsx — fetch('/api/days') → fetch(`${API_BASE}/days`)；删除重复 import Verify（会导致 tsc 编译失败、下次部署挂）
3. src/utils/turnstileLoader.ts — FALLBACK_URL 改用 API_BASE

### 验证
- tsc -b 通过（exit 0）
- 跨域预检 OPTIONS /api/auth/login → 204，access-control-allow-origin: https://textime.top，allow-headers: Content-Type,Authorization
- 跨域 GET /api/health → 200 JSON，CORS 头正确

### 待用户操作
重新部署前端到 Vercel 即可生效。后端、Cloudflare 无需改动。
